MongoDB

三个基本的概念：

数据库(DataBase)

  • 数据库是一个仓库，在仓库里可以存放集合

    集合(Collection)

  • 集合类似于数组，在集合中可以存放文档
    • 集合和数据库都不需要手动创建。

      文档(Document)

  • 文档是数据库中的最小单位，我们储存和操作的内容都是文档。
    • 在创建文档时，如果文档所在的集合或数据库不存在，则会自动创建数据库与集合。

MacOS X Catalina安装:

  • 进入MongoDB官网
  • 下载Community Server解压到/usr/local，并重命名为mongodb
  • 把 MongoDB 的二进制命令文件目录（安装目录/bin）添加到 PATH 路径中
    • export PATH=/usr/local/mongodb/bin:$PATH
  • 首先我们创建一个数据库存储目录
    • sudo mkdir -p ~/data/db(Catalina下储存容器挂载在根目录有问题)
  • 启动MongoDB服务器
    • sudo mongod --dbpath=~/data/db
  • 另外打开一个terminal，进入数据库客户端
    • cd /usr/local/mongodb/bin
    • ./mongo

数据库的服务器和客户端

  • 数据库的服务器用来保存数据
  • 数据库的客户端是用来操作服务器，对数据进行增删改查的操作。

常用指令(shell下)

  • show dbs显示所有的数据库
  • shou collections显示数据库中所有的集合
  • use <name>进入到指定数据库
  • db表示当前所处的数据库

数据库的CRUD操作（增删改查）

增加（Create）：向数据库中插入文档

  • db.<collection>.insert(doc)向集合中插入文档（json格式）
    • 当插入一条数据时，使用json格式
      • insert({name:'SunwuKong',age:18,gender:'male'})
    • 当插入多条数据时，使用数组:
      • `insert([{name:’沙和尚’,age:’38’,gender:’男’},
              {name:'白骨精',age:'16',gender:'女'},
      {name:'猪蜘蛛精',age:'14',gender:'女'},
      ])`
    • 如果在集合中插入文档时，没有指定_id属性，则数据库会自动为文档添加_id属性作为文档的唯一标识。
  • db.<collection>.insertOne(doc)只能插入一个文档
  • db.<collection>.insertMany(doc)只能插入多个文档

    查找(Read)：在数据库中查找文档

  • db.<collection>.find()查询当前集合中的所有符合条件的文档数组
    • find()中可以接受一个对象作为查询条件
      • {}表示查询所有的文档
    • find().count()查询文档数组的数量
  • db.<collection>.findOne()查询当前集合中的所有符合条件的第一个文档对象
    • 可以直接取对象的属性值db.<collection>.findOne().name

修改(Update)：修改数据库中的文档

  • db.<collections>.update(查询条件,新对象)有两个参数，修改成新对象（默认只修改第一个）

    • db.stus.update({name:'沙和尚'}, {age:28})
    • 默认情况下会使用新对象替换旧对象：只剩下age28了

    • 如果需要修改指定属性而不是替换，需要使用修改操作符

      • $set用来修改文档中的指定属性

    • $unset用来删除文档中的指定属性

  • db.<collections>.updateMany(查询条件,新对象)把所有的符合条件的文档都进行修改
  • db.<collections>.updateOne(查询条件,新对象)把第一个符合条件的文档都进行修改
  • db.<collections>.relaceOne(查询条件,新对象)把指定属性进行替换

删除(Delete):将数据库的文档删除

  • db.<collection>.remove()删除符合条件的所有文档（默认）
    • remove()根据条件来删除文档，传递条件的方式和find()一样
    • 如果第二个参数传递一个True则只会删除一个
    • 如果只穿一个空对象，则清空集合（性能较差）
      • db.<collection>.drop()清空集合（性能较好）
  • db.<collection>.deleteOne()删除一个文档
  • db.<collection>.deleteMany()删除多个文档
  • db.dropDatabase()删除数据库
